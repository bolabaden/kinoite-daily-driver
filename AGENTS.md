- Keep Windows/WSL2/WSLg-only explanation in the single canonical doc `config/wsl2/README.md`; keep universal atomic provisioning in **`GETTING_STARTED.md`** (no WSL-only sections in that funnel).
- Onboarding (`GETTING_STARTED.md`): one shared step sequence for every install path; mention WSL2, VM, or imported-rootfs differences only in short per-step CAUTION blocks when behavior actually changes. ## Learned Workspace Facts

- Fedora Kinoite is provisioned declaratively via `config/rpm-ostree/layers.list`, `config/flatpak/*.list`, and `scripts/apply-atomic-provision.sh` (see `GETTING_STARTED.md`).
- WSL2/WSLg: **[config/wsl2/README.md](config/wsl2/README.md)** = narrative; **[scripts/README.md#wsl2-and-wslg](scripts/README.md#wsl2-and-wslg)** = catalog of `scripts/wsl2/*` (no README inside `scripts/wsl2/`). Non-WSL setups follow `GETTING_STARTED.md` and normal desktop login.
- Cross-repo plan references may use KotOR.js `.cursor/plans/kinoite_wsl_workspace_ec9c3c8b.plan.md`; local coverage/verify scripts are written for that plan name. **Pre-push / agent sweep:** [scripts/verify-repo-health.ps1](scripts/verify-repo-health.ps1) runs [check-md-links.ps1](scripts/check-md-links.ps1) and [verify-plan-frontmatter-coverage.ps1](scripts/verify-plan-frontmatter-coverage.ps1) in one go (stops on first failure; set `KOTOR_PLAN_PATH` if the plan is not at a default KotOR.js path). **Editor:** [.vscode/tasks.json](.vscode/tasks.json) — *Run Task* **Kinoite: verify repo health** (full) or **Kinoite: check markdown links (only)** when the KotOR plan is unavailable.
- The repo uses `.gitattributes` with `* text=auto eol=lf` so text files stay LF in the index. **[`.vscode/extensions.json`](.vscode/extensions.json)** recommends the Microsoft **PowerShell** extension for editing the many `*.ps1` files.
- **CI (GitHub):** [`.github/workflows/markdown-link-check.yml`](.github/workflows/markdown-link-check.yml) runs [check-md-links.ps1](scripts/check-md-links.ps1) on push/PR. Full [verify-repo-health.ps1](scripts/verify-repo-health.ps1) (75 KotOR `id`s) stays **local** — the plan file is not checked out in that workflow. **[`.github/dependabot.yml`](.github/dependabot.yml)** (weekly) proposes version bumps for GitHub Actions used in workflows.
- **Secrets plane:** Wi-Fi/VPN/credentials never belong in git — templates under `config/network/`, docs in `config/secrets/README.md`, real files in gitignored `host-local/` (see `GETTING_STARTED.md` step 3).
